Aggregates Q-values from all 8 agents into a final trading action using quantum-inspired voting. Each agent "votes" BUY/SELL based on their Q-value argmax. The system uses confidence-weighted majority voting with deadlock detection, diversity injection, and tie-breaking mechanisms.

Voting & Consensus Systems
Dual voting architecture: Action-level voting for trading decisions + Gradient-level voting for training optimization
8× QuantumAgent Q-values
→ AVN (AttentionVotingNetwork)
→ QuantumVotingSystem
→ Final BUY/SELL Signal

Stage 1: Each of 8 agents produces Q-values [Q_buy, Q_sell] from their timeframe-specific state.
Stage 2: AVN uses multi-head self-attention to learn agent reliability weights + uncertainty quantification.
Stage 3: QuantumVotingSystem uses confidence-weighted majority voting with deadlock detection.
Stage 4: Final signal published via Ably/Redis with pullback logic and diversity enforcement.

Strategy 1: Standard TD
Strategy 2: Conservative
Strategy 3: Aggressive
→ VotingEnsembleTrainer
→ Weighted Gradient Average

Multiple training strategies compute separate gradients, then vote on the optimal update direction. Gradient similarity is measured, and a weighted average is applied. Strategy weights are updated based on loss reduction.
